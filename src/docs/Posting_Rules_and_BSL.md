# Правила проведения и BSL-заготовки

## Общие проверки
- `Quantity > 0`, `Price >= 0`.
- В расходных документах `PartyDocument` обязателен.
- Запрет отрицательных остатков по `ResponsiblePerson + BSOItem + PartyDocument`.

## Движения по регистрам
- `BSOReceipt`: приход в `BSOBalances` и `BSOParties`.
- `BSOIssue`: расход со склада, приход на МОЛ.
- `BSOUsage`: расход у МОЛ по партии, обновление статуса номеров.
- `BSOReturn`: расход у МОЛ, приход на склад.
- `BSOWriteOff`: расход у МОЛ, статус `Damaged/Lost`.

## BSL-фрагменты
```bsl
Функция CalculateAmount(Количество, Цена) Экспорт
	Возврат Окр(Количество * Цена, 2);
КонецФункции
```

```bsl
Процедура ValidateQtyPrice(Количество, Цена, Отказ, ТекстОшибки) Экспорт
	Если Количество <= 0 Тогда
		Отказ = Истина;
		ТекстОшибки = "Количество должно быть больше нуля.";
		Возврат;
	КонецЕсли;
	Если Цена < 0 Тогда
		Отказ = Истина;
		ТекстОшибки = "Цена не может быть отрицательной.";
	КонецЕсли;
КонецПроцедуры
```
