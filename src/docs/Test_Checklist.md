# Чек-лист тестирования и приемки

## Сквозной сценарий
1. `BSOReceipt`: две партии одного вида с разной ценой.
2. `BSOIssue`: выдача МОЛ по партиям.
3. `BSOUsage`: использование части партии.
4. `BSOReturn`: возврат неиспользованных.
5. `BSOWriteOff`: списание испорченных/утраченных.
6. `BSOInventory`: фиксация расхождений.

## Обязательные негативные проверки
- Запрет дублей `ExternalBlankCode`.
- Запрет дублей `Series+Number` в рамках `BSOType`.
- Запрет расхода сверх остатка по `ResponsiblePerson + PartyDocument`.
- Запрет пересечения диапазонов.
- Запрет повторного использования номера.

## Критерии приемки
- Поддержаны БСО с серией/номером и без них.
- Партионный учет обязателен в расходных операциях.
- Цена хранится с точностью 4 знака.
- Все 4 отчета выводят `ExternalBlankCode`.
